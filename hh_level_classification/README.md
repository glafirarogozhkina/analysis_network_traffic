# IT Developer Level Classification (hh.ru)

## Описание задачи

Цель работы — проверить жизнеспособность идеи автоматического определения
уровня IT-разработчика (junior / middle / senior) по данным резюме hh.ru.

Работа выполнена как proof of concept (PoC).

---

## Данные

Используется датасет резюме hh.ru.

Из исходных данных были выделены резюме IT-разработчиков на основе
названия желаемой должности.

---

## Формирование целевой переменной

Целевая переменная `level` формируется эвристически:
- по ключевым словам в названии должности (junior / middle / senior)
- по опыту работы:
  - < 2 лет — junior
  - 2–5 лет — middle
  - > 5 лет — senior

## Модели

Были обучены и сравнены несколько моделей:

### 1. Logistic Regression (числовые признаки)
Признаки:
- зарплата
- возраст
- опыт работы

### 2. Logistic Regression + TF-IDF
Дополнительно использованы текстовые признаки,
извлечённые из названия должности с помощью TF-IDF.

### 3. RandomForestClassifier
Использованы числовые признаки:
- зарплата
- возраст
- опыт работы

---

## Оценка качества

Качество моделей оценивалось с помощью classification report
(precision, recall, F1-score по классам).

Лучший результат показала модель RandomForestClassifier:

- accuracy ≈ 0.92
- macro-F1 ≈ 0.87

---

## Выводы

- Задача автоматического определения уровня разработчика является решаемой.
- Качество модели существенно зависит от используемых признаков.
- Текстовые признаки играют важную роль.
- Основные источники ошибок:
  - дисбаланс классов
  - эвристическая разметка целевой переменной
  - шум в данных резюме

Работа подтверждает жизнеспособность идеи и может быть расширена
за счёт более точной разметки и использования текстов навыков и опыта.
